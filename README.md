# GitHub 极简工作台

一个极简的 GitHub 仓库管理工具，提供本地与云端代码的智能同步比对。

## ✨ 功能特性

### 🔄 自动同步
- **SHA 值精准比对**：通过提交 SHA 值精确判断本地与云端差异
- **自动检测变更**：实时监控本地文件修改状态
- **一键推送/拉取**：根据比对结果自动执行推送或拉取操作

### 📦 仓库管理
- **云端仓库列表**：显示 GitHub 账户下所有仓库及最后更新时间
- **本地项目管理**：展示本地已拉取的项目及其同步状态
- **新建云端仓库**：支持创建公开或私有仓库
- **重建已删除仓库**：当云端仓库被删除时，可从本地重建并推送

### 🛠️ 快捷操作
- **一键打开 VS Code**：快速启动 VS Code 打开项目
- **快速打开文件夹**：直接打开项目所在目录
- **彻底删除本地项目**：完全移除本地项目文件夹

### 🌐 网络支持
- **SingBox 代理**：内置 sing-box 支持，通过 VLESS 节点代理访问 GitHub
- **自动启动**：配置节点后，程序启动自动开启代理
- **HTTP 代理**：sing-box 同时提供 SOCKS5 和 HTTP 代理

### 🖥️ 界面特性
- **现代化 UI**：简洁美观的 Web 界面
- **系统托盘图标**：后台运行时显示托盘图标
- **实时日志控制台**：终端风格的日志显示区域
- **Toast 提示**：友好的操作反馈

## 📋 系统要求

- Windows 10 22H2+
- Python 3.12+
- 网络连接（用于 GitHub API 访问）

## 🚀 安装与运行

### 1. 下载 sing-box

双击运行 `下载sing-box.bat`，自动从 GitHub 下载 sing-box.exe 到程序目录。

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 启动应用

```bash
python app.py
```

启动后会自动打开浏览器访问 `http://127.0.0.1:<端口号>`，同时在系统托盘显示图标。

### 4. 打包为 exe（可选）

```bash
pip install pyinstaller
python build.py
```

打包完成后在 `dist\GitHub推送工具\` 目录下找到可执行文件。

## ⚙️ 配置说明

### GitHub Token

1. 打开设置面板（点击右上角 ⚙️ 按钮）
2. 输入 GitHub Personal Access Token（需要 `repo` 权限）
3. 点击"保存 Token"
4. 可点击"测试连接"验证 Token 是否有效

**如何获取 Token：**
1. 登录 GitHub → Settings → Developer settings
2. 选择 Personal access tokens → Tokens (classic)
3. 生成新 Token，勾选 `repo` 权限

### VLESS 代理节点

1. 在设置面板中粘贴 `vless://` 链接
2. 保存后，程序启动时自动通过 sing-box 启动代理
3. 无需手动配置代理软件

## 🔧 使用指南

### 首次使用

1. 双击 `下载sing-box.bat` 获取代理工具
2. 启动应用后，点击右上角 ⚙️ 按钮打开设置
3. 配置 GitHub Token 和 VLESS 节点
4. 点击 ☁️ "刷新云端" 获取云端仓库列表

### 日常同步

1. 点击本地项目的"智能比对同步"按钮
2. 系统会自动检测：
   - 本地有修改 → 弹窗确认推送
   - 云端较新 → 弹窗确认拉取覆盖
   - 已同步 → 显示"已最新"

### 新建项目

1. 点击 ➕ "新建上云" 按钮
2. 输入仓库名称
3. 选择是否私有
4. 点击"确定创建"

## 🗂️ 数据存储

- 数据库文件：`manager_data.db`（SQLite）
- 本地项目：存储在应用所在目录下
- 配置项：GitHub Token、VLESS 节点链接

## ⚠️ 注意事项

1. **Token 安全**：Token 本地存储，请勿泄露
2. **代理配置**：如遇网络问题，请确认 VLESS 节点配置正确且 sing-box.exe 存在
3. **项目路径**：本地项目存储在应用同级目录，避免移动应用位置
4. **数据备份**：重要项目建议定期备份

## 🐛 常见问题

**Q: 无法获取云端仓库列表？**
A: 检查 GitHub Token 是否正确配置，代理是否正常运行。

**Q: 同步失败？**
A: 确认本地项目是有效的 Git 仓库，且 Token 具有相应权限。

**Q: 系统托盘图标不显示？**
A: 部分系统可能需要重启应用或检查托盘图标设置。

**Q: 代理启动失败？**
A: 确认 sing-box.exe 在程序目录下，且 VLESS 节点链接格式正确。

## 📝 开发说明

本工具使用以下技术栈：
- **后端**：Flask + dulwich（纯 Python Git 实现）
- **前端**：原生 HTML/CSS/JavaScript
- **存储**：SQLite
- **代理**：sing-box（VLESS 协议）
- **系统集成**：pystray（系统托盘）、Pillow（图标）
